# Contributing to Jira Helper

Thank you for your interest in contributing to Jira Helper!

## How to Contribute

- Fork the repository and create your branch from `main`.
- Make your changes, following the project’s code style and conventions.
- **Update the `CHANGELOG.md`:**  
  Every code change, feature, bugfix, or notable update MUST include a corresponding entry in `CHANGELOG.md` under the "Next" section.
- Open a pull request and ensure the PR description clearly summarizes your changes.

## Changelog Entry Policy

- Add your entry under the "Next" section in `CHANGELOG.md`.
- Use concise, clear language.
- Indicate the type of change: Added, Changed, Fixed, Removed, etc.
- If your change relates to an issue or PR, reference it.
    - This should be a commit hash as `([reference](link))`

**Example:**

```
### Added

- New filter for issues table (#123)

### Fixed

- Corrected version display in settings (#130)
```

## Agents and Automation

If you’re using a bot, automation, or Copilot agent, please ensure it follows the same changelog update policy.  
Agents should also consult [AGENTS.md](AGENTS.md) if present for additional instructions.

## Pull Request Checklist

- [ ] My change includes a `CHANGELOG.md` entry.
- [ ] Code follows the repository’s style guidelines.
- [ ] I’ve referenced relevant issues/PRs in my changelog entry.

Thank you for helping improve Jira Helper!
